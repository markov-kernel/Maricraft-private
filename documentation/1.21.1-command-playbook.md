# Minecraft Java 1.21.1 Commands Playbook (NeoForge‑compatible)

Scope: Java Edition 1.21.1 (released 2024‑08‑08). NeoForge does not change vanilla
command syntax; everything here works the same on NeoForge 1.21.1 unless a mod explicitly
alters behavior.

Use this as a practical, copy‑pasteable reference with pitfalls called out. Where
behavior often confuses players, links and short notes are provided.

---

## Quick Highlights (What changed in 1.21.x)

- Items use Data Components (structured fields) instead of legacy ad‑hoc NBT on items.
  Anywhere an “item stack” is accepted, you can write `item_id[component=value,...]`.
  Example: `/give @p minecraft:bow[enchantments={power:5}] 1`.
- Enchantments are a component (`enchantments`) with an id→level map.
- Attribute modifiers live in the `attribute_modifiers` component with fields
  `type` (attribute id), `amount`, `operation`, `slot`, and `id` (unique string).
- Text components for names/lore in 1.21.1 are JSON text (embedded as strings). In 1.21.5+
  these move to SNBT object form, but that is not 1.21.1.
- Big build ops (`/fill`, `/clone`) are bounded by gamerule `commandModificationBlockLimit`.

References: Data Components overview; specific pages for enchantments, attribute_modifiers,
coordinates, execute, fill/clone limits.

---

## Command Length and Limits

- Chat command length: client truncates at 256 characters. Use command blocks or functions
  for longer sequences.
- Fill/Clone size: governed by gamerule `commandModificationBlockLimit` (not a hard 32,768 in
  every world). If a command lags or fails, reduce dimensions or raise the gamerule (admins only).

Examples:
```
# gamerule (server/op only)
/gamerule commandModificationBlockLimit 32768
```

## Coordinates and Facing

- Absolute: `X Y Z`
- Relative: `~dx ~dy ~dz` (e.g., `~ ~1 ~-5`)
- Local (entity‑relative): `^left ^up ^forward` (e.g., `^0 ^0 ^10`) aligned to the executor’s
  rotation.
- Do not mix coordinate types within the same XYZ triplet.
- Bare `~` and `^` are allowed and mean 0; `^0` and `~0` are equivalent.

Common wrapper (place/build D blocks ahead in your look direction):
```
execute positioned ^0 ^0 ^10 run <your command using ~ or ^>
# Eye height origin, useful for rays/line‑of‑sight:
execute anchored eyes positioned ^0 ^0 ^10 run <...>
```

## Target Selectors (essentials)

- Bases: `@p` (nearest player), `@a` (all players), `@e` (all entities), `@s` (self), `@r` (random).
- Filters: `type=`, `distance=`, `sort=nearest|furthest|random|arbitrary`, `limit=`, `scores=`, etc.

Examples:
```
@e[type=zombie,distance=..10,limit=1,sort=nearest]
@p[scores={kills=10..}]
```

## Data Components (items) in 1.21.1

### General Item Stack Form
```
<namespace:id>[component1=value,component2=value,...] [count]
# Example
/give @p minecraft:netherite_sword[enchantments={sharpness:5}] 1
```

### Attribute Modifiers (melee damage/speed)

- Component: `attribute_modifiers=[{...}, ...]`
- Fields per entry:
  - `type`: attribute id (1.21.1 uses `minecraft:generic.*`, e.g., `minecraft:generic.attack_damage`)
  - `amount`: number
  - `operation`: `add_value` | `add_multiplied_base` | `add_multiplied_total`
  - `slot`: e.g., `mainhand`, `offhand`, `head`, `chest`, `legs`, `feet`
  - `id`: unique string identifier for the modifier

Examples:
```
# One‑shot damage
/give @p netherite_sword[
  attribute_modifiers=[
    {type:"minecraft:generic.attack_damage",amount:2048,operation:"add_value",slot:"mainhand",id:"playbook:ad"}
  ]
] 1

# Faster attack speed (optional)
/give @p netherite_sword[
  attribute_modifiers=[
    {type:"minecraft:generic.attack_speed",amount:1024,operation:"add_value",slot:"mainhand",id:"playbook:as"}
  ]
] 1
```

Version note: In 1.21.2+ Mojang renamed attribute ids (e.g., `minecraft:attack_damage`).
For 1.21.1, use the `minecraft:generic.*` form.

### Enchantments

Component: `enchantments={id1:level,id2:level,...}`
```
/give @p diamond_sword[enchantments={sharpness:5,fire_aspect:2,unbreaking:3}] 1
```

### Potion / Tipped Arrow Effects

Component: `potion_contents={potion:"minecraft:awkward",custom_effects:[{id:"minecraft:poison",amplifier:3,duration:200},...]}`
- Durations are in ticks (20 ticks = 1 second).
- `amplifier:0` is Level I, `1` is Level II, etc.

```
# 64 “venom” arrows: Poison IV for 10s + Weakness II for 30s
/give @p tipped_arrow[
  potion_contents={
    custom_effects:[
      {id:"minecraft:poison",amplifier:3,duration:200},
      {id:"minecraft:weakness",amplifier:1,duration:600}
    ]
  }
] 64
```

### Pre‑Charged Crossbows
```
/give @p crossbow[
  charged_projectiles=[
    {id:"minecraft:tipped_arrow",count:1,components:{
      "minecraft:potion_contents":{
        custom_effects:[{id:"minecraft:poison",amplifier:3,duration:200}]
      }
    }}
  ]
] 1
```

### Names and Lore (1.21.1 JSON text)

- `custom_name` and `lore` take JSON text values in 1.21.1, usually embedded as strings.
```
/give @p stick[
  custom_name='{"text":"Admin Wand","color":"gold","italic":false}',
  lore=[
    '{"text":"Right‑click: secrets","color":"dark_purple"}'
  ]
] 1
```

### Unbreakable
```
/give @p netherite_pickaxe[unbreakable={}] 1
```

### Custom Model Data (resource pack integration)

- 1.21+ supports richer custom_model_data formats; consult your pack docs.

## Core Commands You’ll Use Often

### /give
```
/give <targets> <item_stack> [count]
```
Examples:
```
# Oneshot sword (1.21.1‑correct attribute ids)
/give @p netherite_sword[
  attribute_modifiers=[{type:"minecraft:generic.attack_damage",amount:2048,operation:"add_value",slot:"mainhand",id:"playbook:ad"}]
] 1

# Same, with Fire Aspect via /enchant (often shorter than embedding)
/enchant @p minecraft:fire_aspect 10
```

### /item (replace/modify)
```
/item replace entity <targets> <slot> with <item_stack> [count]
/item modify entity <targets> <slot> <data_modifier>
```
Example:
```
/item replace entity @p weapon.mainhand with netherite_sword[unbreakable={}] 1
```

### /effect
```
/effect give <targets> <effect> [seconds] [amplifier] [hideParticles]
```
Examples:
```
# Brutal DoT on nearby non‑players
/effect give @e[type=!player,distance=..5,limit=1,sort=nearest] minecraft:wither 10 4 true
/effect give @e[type=!player,distance=..5,limit=1,sort=nearest] minecraft:poison 10 3 true
```

### /execute (patterns)
```
# Place 10 blocks ahead of your eyes
execute anchored eyes positioned ^0 ^0 ^10 run setblock ~ ~-1 ~ minecraft:stone

# Build a box anchored D blocks ahead (world‑aligned)
execute positioned ^0 ^0 ^10 run fill ~-5 ~-1 ~-5 ~5 ~3 ~5 minecraft:stone_bricks hollow
```

### /scoreboard (minimal)
```
# Create objective to track damage dealt
/scoreboard objectives add dealt minecraft.custom:minecraft.damage_dealt

# Read/limit by score in selectors
@a[scores={dealt=1..}]
```

### /data (debug and storage)
```
/data get entity @s SelectedItem
/data modify storage playbook:store last set from entity @s SelectedItem
```

## Building at Speed: Fill/Clone Wrappers and Safety

- Wrap large edits in `execute positioned ^0 ^0 ^D` to build in front of you.
- Keep single‑command volumes reasonable for performance. If laggy, split along the
  longest axis and send multiple `/fill` commands.
- Raise `commandModificationBlockLimit` only when you understand the server impact.

Examples (site prepare + base):
```
# Clear a 35×21×35 air pocket 10 blocks ahead
execute positioned ^0 ^0 ^10 run fill ~-17 ~-5 ~-17 ~17 ~15 ~17 minecraft:air
# Stamp a grass floor at Y‑1 (relative to the origin above)
execute positioned ^0 ^0 ^10 run fill ~-17 ~-1 ~-17 ~17 ~-1 ~17 minecraft:grass_block
```

## Common Pitfalls (1.21.1)

- “Unknown registry key … minecraft:attack_damage” while using attribute_modifiers → In 1.21.1
  the attribute id must be `minecraft:generic.attack_damage` (and similarly for attack_speed).
  The short ids (e.g., `minecraft:attack_damage`) are for 1.21.2+.
- Names/lore format → In 1.21.1 you embed JSON text as strings. If you paste SNBT objects for
  text fields (the 1.21.5+ style), they won’t parse here.
- Mixing coordinate types → Never combine `~` and `^` in the same XYZ triplet.
- Chat length → Commands over 256 characters get truncated; split or use blocks/functions.

## Version Notes (future‑proofing)

- 1.21.2+: attribute ids no longer use `generic.` (e.g., `minecraft:attack_damage`). Update
  your commands accordingly if you upgrade the world/server.
- 1.21.5+: many item text/lore and some other components accept SNBT object forms directly and
  add `tooltip_display`. Retain the 1.21.1 JSON‑string approach until you actually move up.

---

## Troubleshooting Checklist

- Attribute errors → confirm the attribute id is `minecraft:generic.*` (1.21.1), operation is one
  of `add_value|add_multiplied_base|add_multiplied_total`, and slot matches where you equip it.
- “Malformed component …” → check commas and quotes inside bracket components; JSON text values
  must be valid JSON strings when embedded.
- Nothing appears to happen → check if you built with `execute positioned ^0 ^0 ^D` into a wall
  or underground; adjust D or Y.
- Command too long → shorten names/lore, move some data into `/enchant` or split steps.
